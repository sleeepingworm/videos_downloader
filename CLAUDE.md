# 视频下载工具使用规则

本项目是一个 YouTube 视频下载工具。当用户给出 YouTube URL 并表达下载意图时，请按以下流程操作。

## 前置条件

所有命令在项目根目录下运行，使用 `python -m scripts.video_fetch` 方式执行。

## 交互流程

1. **解析视频信息**：运行 `python -m scripts.video_fetch inspect <url>`，获取视频元信息。
2. **展示下载计划**：向用户展示：
   - 视频标题、平台、时长
   - 目标保存目录
   - 磁盘剩余空间
   - 可选质量列表（最多 5 个选项），每项标注分辨率、容器、预计大小
3. **询问确认**：让用户选择质量或取消。
4. **执行下载**：用户确认后，运行 `python -m scripts.video_fetch download <url> --format <format_id> --output-dir <path>`。
5. **展示结果**：告知用户保存路径和实际文件大小。

## 关键规则

- 绝不能静默下载。任何下载前都必须展示信息并等待用户确认。
- 绝不能批量下载播放列表。如果检测到多 P 或播放列表，先询问用户范围。
- 磁盘空间不足时不能继续下载。
- 如果工具返回 `unsupported_url`：告知用户该 URL 不支持。
- 如果工具返回 `dependency_missing`：告知用户缺少依赖，给出安装命令。
- 如果工具返回 `auth_required`：告知用户此视频需要登录，当前版本不支持。
- 如果工具返回 `download_failed`：展示错误摘要，不自动重试。
